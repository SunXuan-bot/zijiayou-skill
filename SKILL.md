# 自驾路线规划Skill

> 基于路线库 + 高德地图API，为用户规划自驾行程（含一键导航链接）

---

## ⚠️ 首要原则：先问需求，再给方案

**严禁在未确认需求前直接输出完整路书！**

用户触发后，先执行「需求确认三问」，等用户回答后再规划。

---

## 需求确认三问（触发后必问）

```
好的，我来帮你规划！
先问几个关键问题，帮我缩小范围：

① 你们是几个人出行？有老人或小孩吗？
② 有没有特别想去的城市或景区？
③ 计划玩几天？预算大概多少？
```

**有了答案后再调用工具、生成方案。**

---

## 触发词

- "帮我规划"、"帮我设计路线"、"自驾推荐"
- "从XX出发，X天"、"X日自驾"
- "去XX自驾，X人"
- "带X自驾推荐"

## 输入参数（必须识别）

| 参数 | 说明 | 来源 |
|------|------|------|
| 出发地 | 城市名 | 用户说"从XX出发" |
| 目的地 | 城市/省份 | 用户说"去XX" |
| 天数 | 数字 | 用户说"X天" |
| 人数 | 数字（可选） | 用户说"X人" |
| 人群类型 | 亲子/情侣/老人/朋友/团队（可选） | 用户描述推断 |
| 预算 | 高/中/低（可选） | 用户描述 |
| 偏好 | 自然风光/人文/美食/休闲（可选） | 用户描述 |

**注意：参数不全时，先用高德API查，再结合路线库回答，不要让用户等。**

## 数据来源

### 路线库
- **大自驾网数据**：`data/routes_travel.json`（865条全国路线，含完整行程）
- **武汉驾到实际活动**：`data/routes_wuhan.json`（18条，已执行验证）

### 地图工具
- **高德地图MCP Server**：通过 mcporter 调用
- 服务名：`amap`
- 调用示例：`mcporter call amap.maps_direction_driving_by_address origin_address="武汉" destination_address="无锡"`

## 工作流程

```
用户输入 → 解析参数 → 判断是否有完整参数
    ↓
参数完整 → 搜路线库 → 调用高德API → 生成路书
    ↓
参数不全 → 调用高德API获取基础信息 → 基于路线库经验回答
```

### 步骤1：解析用户输入

识别出发地、天数、人群类型，优先从路线库匹配。

### 步骤2：调用高德API（必要步骤）

**必须调用的工具：**

```bash
# 驾车路线规划（必调）
mcporter call amap.maps_direction_driving_by_address \
  origin_address="出发地" \
  destination_address="目的地"

# 目的城市景点搜索（必调）
mcporter call amap.maps_text_search \
  keywords="目的地 景点" \
  city="目的地城市"

# 天气预报（出行前必调）
mcporter call amap.maps_weather city="目的城市"
```

### 步骤3：结合路线库生成方案

从 `data/routes_travel.json` 匹配相同目的地、天数相近的路线作为参考。

### 步骤4：生成输出

按标准格式输出路书（见下方）。

---

## 输出格式

### 导航链接生成（每个景点必须带）

**获取方式：**
1. `maps_text_search` 搜景点名 → 获取POI ID
2. `maps_search_detail` 用POI ID → 获取经纬度坐标
3. 生成链接：`https://uri.amap.com/navigation?to=<经度>,<纬度>,<景点名>&mode=car&src=武汉驾到`

**链接格式：**
```
📍【景点名】 [导航去这里](https://uri.amap.com/navigation?to=经度,纬度,景点名&mode=car&src=武汉驾到)
```

**重要：每个景点都必须生成导航链接，不能省略。**

## Skill约束规则

### 必须遵守

1. **公里数必须来自高德API**，不得自行估算
2. **景点必须实际存在**，不得虚构景点
3. **住宿推荐必须标注价格区间**，不得写"高档酒店"等模糊词
4. **出发前必须查天气预报**，并写入行程
5. **每个景点必须生成高德导航链接**，不得遗漏
6. **自驾段必须包含高速名称和过路费估算**

### 禁止行为

1. 禁止输出没有具体数据的泛泛推荐（如"沿途风景很美"）
2. 禁止虚构不存在的景点或餐厅
3. 禁止在用户未告知预算时擅自设定价格区间
4. 禁止输出时省略里程数和驾车时间
5. **禁止省略导航链接**

### 输出前自检

- [ ] 总里程（公里数）已填
- [ ] 驾车时间已填
- [ ] 途经高速名称已填
- [ ] 过路费估算已填
- [ ] 目的地天气已查
- [ ] 景点名称具体
- [ ] **每个景点都有导航链接**
- [ ] 住宿有价格区间
- [ ] 费用参考含自驾成本明细

## 路线库字段说明（data/routes_travel.json）

```json
{
  "线路名称": "...",
  "天数": 5,
  "省份": "贵州",
  "线路亮点": "...",
  "完整行程": "D1: ...\nD2: ..."
}
```

### 路线库筛选逻辑

1. **按天数过滤**：优先选天数完全匹配的路线；无完全匹配时选±1天的
2. **按目的地过滤**：优先选省份匹配的
3. **按人群标签过滤**：有标签的优先返回

---

## 人群类型判断规则

| 关键词 | 人群类型 |
|--------|---------|
| "孩子"、"小朋友"、"亲子"、"带娃" | 亲子 |
| "父母"、"老人"、"带爸妈" | 养老 |
| "情侣"、"夫妻"、"蜜月" | 情侣 |
| "朋友"、"闺蜜"、"兄弟" | 朋友 |
| "公司"、"团建"、"同事" | 团队 |

## 常见路线参考（备用）

当路线库无匹配时，使用以下经验值：

| 天数 | 单日合理里程 | 住宿建议 |
|------|------------|---------|
| 2日 | ≤400km/天 | 目的地1晚 |
| 3日 | ≤350km/天 | 目的地2晚，或中途换1次 |
| 4日 | ≤300km/天 | 最多换2次酒店 |
| 5日+ | ≤300km/天 | 每天不换超过1次 |

---

## 错误处理

- **高德API调用失败**：降级为"基于路线库经验规划"，并说明"以下为参考数据，实际路线请以导航为准"
- **路线库无匹配**：搜索相似目的地路线，并在输出中标注"参考路线"
- **参数完全缺失**（如只说"推荐一条路线"）：先询问出发地和天数，不自行假设
